⭐ 一、总体设计思路与核心优势

这份配置的核心目标是：

1. 自动化程度高（节点自动筛选、自动测速、自动健康检查）

三份订阅源（P1 / P2 / P3）均启用了自动健康检查与正则过滤，可确保无效节点、广告节点、过期节点自动排除，避免杂乱。

2. 分流体系完整且精细（媒体 / 社交 / 特定应用逐项选择）

常用服务如 YouTube、Google、Telegram、Twitter、Netflix、TikTok 等全部单独分组，允许按场景选择最佳地区节点。

3. 地区节点池智能测速（url-test 自动动态优选）

HK、TW、JP、SG、US 等地区全部采用 url-test 自动延迟检测，在每次切换时自动挑选最优节点。

4. DNS 采用 Fake-IP + 多重回退，确保速度与兼容性平衡

Fake-IP + 国内 DNS 分流 + fallback 国际 DNS，兼顾速度与跨境解析的稳定性。

5. TUN 接管系统流量，适合全局代理与本地应用的最大兼容

开启 TUN、自动路由、自动 Hijack，本地应用无需额外配置即可走代理。

6. 完整注释、结构清晰、逻辑严谨，长期使用维护成本低

适合从小白到进阶用户长期维护，相比网上大部分无注释模板更可靠。

⭐ 二、配置主要模块说明
1. proxy-providers（订阅管理 + 节点过滤）

配置启用了三份订阅（P1/P2/P3），并带：

健康检查（health-check）

节点名称前缀标记（additional-prefix）

正则过滤过滤掉无用节点（广告/返利/售卖/机场等字样）

优点：
有效减少垃圾节点，使节点列表干净且易管理。

2. proxies（额外自定义节点）

加入了手动自定义的港台线路（SS + IEPL 混合）。
这些节点可作为“自有的稳定线路”，在分流中随时可选。

优点：
确保有至少一部分节点不依赖订阅服务商，提升稳定性。

3. 基础设置（mixed-port / ipv6 / 指纹 / UI）

开启 IPv6 支持

使用 Chrome 指纹提升网站兼容

外部控制器端口与 UI 一并配置好

优点：
即插即用，适配性强，可使用 MetaCubeX 的 WebUI 远程管理。

4. 流量识别 Sniffer

启用 HTTP / TLS / QUIC 流量识别，并支持 override-destination。

优点：
自动解析目标域名，提高分流命中率，减少误走直连。

5. TUN 模式（系统接管）

全系统流量接管（auto-route）

自动接口识别

Fake-IP DNS 劫持

优点：
适合 Windows/Mac 使用者，所有应用自动走代理，无需逐个设置。

6. DNS 模块（Fake-IP + 国内外分流）

关键特性：

Fake-IP 模式

国内域名走国内 DNS（120.53.53.53 / 223.5.5.5）

国外域名走 Cloudflare/Google

fallback 策略清晰

针对字节系/OceanEngine 单独优化

优点：
本地服务兼容更好、国际解析速度快、跨境网站更稳定。

7. proxy-groups（分流策略组）

这是整个配置的亮点之一：

⬆️ （1）总控组

Proxy（总代理选项）

Final（最终走向）

⬆️ （2）常用应用分流

每类应用可选择最佳地区：

YouTube

Google

Telegram

Twitter

TikTok

Netflix

Github

Spotify

Instagram

OpenAI（ChatGPT 专组）

优点：
真正做到精细分流，不同应用走最适合的节点，提升体验。

8. 地区节点自动测速（HK / TW / JP / SG / US…）

采用 url-test + filter 匹配地区关键字：

自动指向最优延迟节点

定期测速

图标美化（便于 UI 识别）

优点：
自动选择最优节点，无需手动切换。
